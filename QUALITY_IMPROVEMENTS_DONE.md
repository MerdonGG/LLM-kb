### Этап 2: Оптимизация параметров поиска ⭐⭐⭐⭐⭐
**Эффект:** +10-20% качества

**Что изменено:**
1. **Увеличено k:** 8 → 12 (больше кандидатов для фильтрации)
2. **Добавлена фильтрация по релевантности:** distance < 1.5
3. **Добавлена дедупликация:** убираем похожие фрагменты
4. **Улучшен fallback:** если ничего не нашли, берём топ-3

**Алгоритм:**
```python
1. Извлекаем 12 кандидатов
2. Фильтруем по distance < 1.5 (только релевантные)
3. Убираем дубликаты (сравнение первых 100 символов)
4. Ограничиваем до 8 лучших
5. Если ничего не осталось → берём топ-3 без фильтра
```

### Этап 4: Гибридный поиск (BM25 + векторный) ⭐⭐⭐⭐⭐
**Эффект:** +25-35% качества

**Что это:**
Комбинация двух методов поиска:
- **BM25 (keyword search):** хорош для точных терминов ("SHA-256", "RSA", "AES")
- **Векторный (semantic search):** хорош для концепций ("шифрование", "безопасность")

**Как работает:**
```python
1. Векторный поиск → топ-24 кандидата
2. BM25 поиск → топ-24 кандидата
3. Нормализация скоров (0.0 - 1.0)
4. Комбинирование: score = 0.5 * vector + 0.5 * bm25
5. Сортировка по комбинированному скору
6. Фильтрация и дедупликация
7. Возврат топ-8
```

**Параметр alpha:**
- `alpha = 0.0` → только BM25 (keyword)
- `alpha = 0.5` → баланс (по умолчанию)
- `alpha = 1.0` → только векторный (semantic)

**Преимущества:**
- Находит документы по точным терминам (BM25)
- Находит документы по смыслу (векторный)
- Лучшая релевантность результатов

**Новая зависимость:**
- `rank-bm25` — библиотека для BM25

**Кеширование:**
- BM25 индекс сохраняется в `chroma_db/bm25_index.pkl`
- Загружается при старте (не пересоздаётся каждый раз)
