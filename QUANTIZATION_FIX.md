# Исправление: Правильные названия квантизованных моделей

## Проблема
Модель `qwen2.5:3b-q4_K_M` не найдена в репозитории Ollama.

## Решение

В Ollama квантизованные модели имеют другой формат названий. Вот правильные варианты:

### Формат 1: Через двоеточие (стандартный)
```bash
ollama pull qwen2.5:3b-q4_0
ollama pull qwen2.5:3b-q5_0
ollama pull qwen2.5:3b-q8_0
```

### Формат 2: Через тег
```bash
ollama pull qwen2.5:3b-instruct-q4_0
ollama pull qwen2.5:3b-instruct-q5_0
```

### Формат 3: Проверка доступных тегов
```bash
# Посмотреть все доступные версии модели
ollama show qwen2.5:3b --modelfile
```

## Правильные команды для квантизации

### Шаг 1: Проверить доступные версии

```bash
# На сервере
ssh ivan@192.168.29.241

# Попробовать разные форматы
ollama pull qwen2.5:3b-q4_0
```

Если не работает, попробуйте:

```bash
# Вариант 1: Базовая квантизация
ollama pull qwen2.5:3b-q4

# Вариант 2: С суффиксом _0
ollama pull qwen2.5:3b-q4_0

# Вариант 3: С суффиксом _K_M
ollama pull qwen2.5:3b-q4_K_M

# Вариант 4: Через instruct
ollama pull qwen2.5:3b-instruct-q4
```

### Шаг 2: Альтернативный подход - использовать параметры Ollama

Вместо скачивания квантизованных моделей, можно использовать параметры генерации:

```python
# В backend/main.py
resp = requests.post(
    f"{OLLAMA_URL}/api/generate",
    json={
        "model": model,
        "prompt": prompt,
        "stream": True,
        "options": {
            "num_ctx": 2048,      # Меньше контекста = быстрее
            "num_predict": 512,   # Ограничить длину ответа
            "temperature": 0.7,   # Меньше = быстрее
            "top_k": 40,          # Меньше = быстрее
            "top_p": 0.9,
            "repeat_penalty": 1.1
        }
    },
    timeout=600,
    stream=True
)
```
