# СРОЧНАЯ ОПТИМИЗАЦИЯ СКОРОСТИ

## Проблема
После улучшений качества (гибридный поиск, улучшенный промпт) скорость ответов резко упала.

## Причины замедления

1. **Гибридный поиск** — делает 2 поиска вместо 1 (векторный + BM25)
2. **Увеличенный k** — 12 вместо 8 (больше обработки)
3. **Улучшенный промпт** — длиннее (больше токенов для обработки)
4. **Метаданные** — дополнительная обработка

## Решения (от быстрых к радикальным)

---

### ⚡ РЕШЕНИЕ 1: Уменьшить num_predict (САМОЕ БЫСТРОЕ)
**Время:** 2 минуты
**Эффект:** -30-40% времени
**Потеря качества:** Минимальная (ответы короче)

```python
# В backend/main.py
"options": {
    "num_ctx": 2048,
    "num_predict": 256,  # Было: 512 → Стало: 256
    "temperature": 0.7,
    "top_k": 40,
    "top_p": 0.9,
    "repeat_penalty": 1.1
}
```

---

### ⚡ РЕШЕНИЕ 2: Отключить гибридный поиск
**Время:** 2 минуты
**Эффект:** -20-30% времени
**Потеря качества:** -10-15% (только векторный поиск)

```python
# В backend/main.py, в эндпоинте /ask
# Было:
context, sources = hybrid_retrieve(req.question)

# Стало:
context, sources = retrieve(req.question, k=8)  # Вернуться к простому поиску
```

### ⚡ РЕШЕНИЕ 3: Уменьшить k

