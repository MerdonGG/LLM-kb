# Настройка TinyLlama с огромной векторной базой

## Что изменилось:

### 1. Модель
- **Было:** Qwen 2.5 (3B), Qwen 3 (8B), Llama 3.1 (8B)
- **Стало:** TinyLlama (самая маленькая модель)

### 2. Векторная база
- **Размер чанков:** 1000 символов (было 2000)
- **Перекрытие:** 200 символов (было 400)
- **Макс. фрагментов:** 20 (было 4)

### 3. Ожидаемый эффект
- **Скорость:** +50-70% быстрее
- **Качество:** Более точные ответы за счёт большего контекста

## Доступные модели:

| Модель | Скорость | Описание |
|--------|----------|----------|
| `tinyllama` | ~2-5 сек | Самая маленькая модель (по умолчанию) |
| `qwen2.5:3b` | ~5-10 сек | Быстрая |
| `qwen3:8b` | ~15-25 сек | Качественная |
| `llama3.1:8b` | ~20-30 сек | Лучшая |

---

## Как использовать:

**В интерфейсе** выберите модель из dropdown. По умолчанию используется `tinyllama`.

**Пример запроса:**
```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"question": "Привет", "model": "tinyllama", "stream": true}'
```

---

## Важно:

- При первом запуске векторная база будет пересоздана (может занять несколько минут)
- Меньшие чанки = больше детализации
- Больше фрагментов (20) = больше контекста для ответа
- TinyLlama будет работать быстро даже на CPU
